# 橘猫 — 看见自己的能量 产品需求文档

## 1. 产品概述

极简个人能量管理工具，帮助用户看见自己的能量模式。不是任务管理器，是帮助用户觉察自身能量状态的镜子。
- 核心理念：记录和觉察，不是管理和控制；今天只关心今天；不给内在评判者提供弹药
- 目标用户：需要觉察自身能量模式、减少自我评判的个人

## 2. 核心功能

### 2.1 用户角色
| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 单用户 | 无需注册 | 使用全部功能（本地应用） |

### 2.2 功能模块
1. **今日任务页**：每日任务添加、完成、自动清空
2. **进行中页**：多天事项记录，手动推进
3. **能量记录**：每日3个时间点能量评分
4. **每日一句话**：每日一句话记录
5. **周复盘页**：自动数据汇总 + 手动感受

### 2.3 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 今日任务 | 任务列表 | 显示今日任务，支持添加、完成打勾、输入耗时后消失 |
| 今日任务 | 添加任务 | 输入名称，选耗能等级（充电/普通/消耗），选分类（生存/创作/恢复） |
| 今日任务 | 完成任务 | 打勾 → 输入耗时（默认30分钟）→ 任务消失 |
| 进行中 | 事项列表 | 显示进行中的事项，可添加、标记完成 |
| 进行中 | 推进事项 | 手动添加到今日任务 |
| 能量记录 | 能量评分 | 起床/下午/晚间三个时间点，滑动选择1-10分 |
| 能量记录 | 补录 | 错过时间点可当天补录 |
| 每日一句话 | 文字输入 | 20:00后显示输入框，最多100字 |
| 周复盘 | 自动汇总 | 本周平均能量、能量趋势、任务分类分布、耗能等级分布 |
| 周复盘 | 手动感受 | 3个开放问题：最好的一天、最消耗的一天、一句话总结 |

## 3. 核心流程

### 3.1 今日任务流程
用户打开应用 → 看到今日任务列表 → 点击添加 → 输入任务名称 → 选择耗能等级（默认普通）→ 选择分类（默认生存）→ 任务出现在列表 → 完成时打勾 → 输入耗时（默认30分钟）→ 任务消失 → 当天结束未完成任务自动清空

```mermaid
flowchart TD
    "打开应用" --> "查看今日任务列表"
    "查看今日任务列表" --> "添加任务"
    "添加任务" --> "输入任务名称"
    "输入任务名称" --> "选择耗能等级"
    "选择耗能等级" --> "选择分类"
    "选择分类" --> "任务出现在列表"
    "查看今日任务列表" --> "完成任务"
    "完成任务" --> "打勾"
    "打勾" --> "输入耗时"
    "输入耗时" --> "任务消失"
    "当天结束" --> "未完成任务自动清空"
```

### 3.2 能量记录流程
到时间点 → 显示极简弹窗 → 滑动选择1-10 → 确认保存 → 弹窗消失（可忽略弹窗，当天补录）

```mermaid

    "到时间点" --> "显示极简弹窗"
    "显示极简弹窗" --> "滑动选择1-10"
    "滑动选择1-10" --> "确认保存"
    "确认保存" --> "弹窗消失"
    "显示极简弹窗" --> "忽略弹窗"
    "忽略弹窗" --> "当天补录"
    "当天补录" --> "滑动选择1-10"
```

### 3.3 周复盘流程
用户主动打开周复盘 → 查看自动汇总数据 → 回答3个手动感受问题 → 保存

```mermaid

    "查看自动汇总" --> "耗能等级分布"
    "查看自动汇总" --> "回答手动感受"
    "回答手动感受" --> "保存"
```

## 4. 用户界面设计

### 4.1 设计风格
- **主色调**：深色背景（近黑 #0D0D0D），暖橙色点缀（#F97316 橘猫色），辅以柔和的灰白文字
- **按钮风格**：圆角胶囊按钮，无3D效果，hover时微微发光
- **字体**：使用有温度感的中文友好字体，标题用偏粗体，正文用适中粗细
- **布局风格**：左侧导航栏 + 右侧内容区，大量留白，卡片式内容分组
- **图标/emoji**：耗能等级用 🟢⚪🔴 圆点，整体克制不花哨
- **动画**：极简微动效，任务完成时轻柔消失，弹窗淡入淡出

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|
| 今日任务 | 任务列表 | 深色卡片列表，每项左侧耗能圆点，右侧分类标签，hover微亮 |
| 今日任务 | 添加任务 | 底部浮出面板，输入框 + 三个选项组（耗能/分类），确认按钮 |
| 今日任务 | 完成弹窗 | 居中弹窗，耗时滑块（5-180分钟），确认按钮 |
| 进行中 | 事项列表 | 简洁列表，每项右侧"推到今天"按钮和"完成"按钮 |
| 能量记录 | 评分弹窗 | 极简居中弹窗，1-10滑动条，当前数字大号显示 |
| 每日一句话 | 输入区域 | 简洁文本框，字数计数器，保存按钮 |
| 周复盘 | 汇总数据 | 四个数据卡片横排，下方三个开放问题文本框 |

### 4.3 响应式设计
- 桌面端优先（Mac主要平台）
- 手机端适配：导航栏变为底部Tab，支持单手操作
- 触控优化：按钮/滑块尺寸适配手指操作

### 4.4 数据保留规则
| 数据类型 | 保留策略 |
|----------|----------|
| 已完成每日任务 | 保留7天自动删除 |
| 未完成每日任务 | 当天结束删除 |
| 进行中事项 | 完成后保留30天自动删除 |
| 能量记录 | 永久保留 |
| 每日一句话 | 永久保留 |
| 周复盘 | 永久保留 |

### 4.5 明确不做
- 不做任务库/待办清单、未来任务提醒、番茄钟/时间追踪
- 不做社交/分享、图表/趋势分析、成就系统/连续打卡
- 不做AI分析/建议、云端同步、子任务/任务依赖
- 不做优先级排序、完成率统计
- 不用推送通知（仅应用内温和提示）、不用声音/震动
- 不显示成就/徽章/连续天数
